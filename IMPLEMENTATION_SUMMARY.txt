 2. デプロイフロー概要
================================================================================

【開発環境 (develop ブランチ)】
  git push origin develop
    ↓
  GitHub Actions: test.yml 実行
    ├─ Unit Tests
    ├─ Linting
    └─ Security Scan
    ↓
  GitHub Actions: build-web.yml, build-ios.yml, build-android.yml 並列実行
    ├─ Web: npm run web → Vercel デプロイ (preview)
    ├─ iOS: eas build → TestFlight 提出
    └─ Android: eas build → Google Play 内部テスト版
    ↓
  完了まで: 1-2 時間

【本番環境 (main ブランチ)】
  方法 1: git push origin main
  方法 2: git tag -a v1.0.5 && git push origin v1.0.5
  方法 3: ./scripts/deploy.sh -e production -p all -s
    ↓
  GitHub Actions: test.yml + build-* 全実行
    ├─ Web: Cloudflare Pages デプロイ
    ├─ iOS: TestFlight 提出
    └─ Android: Google Play 内部テスト版
    ↓
  Slack #deployments チャネルに通知
    ↓
  完了まで: 1-2 時間

【ロールバック】
  ./scripts/rollback.sh -p ios    # iOS のみ
  ./scripts/rollback.sh -p all    # 全プラットフォーム
    ↓
  前バージョンを自動検出
  対話的に確認
    ↓
  完了まで: 30-45 分
