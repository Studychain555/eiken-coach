### 2.3 認証フロー

```
【OAuth 2.0 認証フロー】

1. ユーザーが「ログイン」をタップ
   ↓
2. /auth/login エンドポイントへ POST
   {
     "email": "user@example.com",
     "password": "hash_of_password"
   }
   ↓
3. サーバーが JWT トークン返却
   {
     "access_token": "eyJhbGciOiJIUzI1N...",
     "refresh_token": "eyJhbGciOiJIUzI1N...",
     "expires_in": 3600
   }
   ↓
4. 以後のリクエストに Authorization ヘッダで送信
   Authorization: Bearer eyJhbGciOiJIUzI1N...
   ↓
5. トークン有効期限: 1時間
   リフレッシュ時は /auth/refresh で新規トークン取得
```

## 3️⃣ データ保護 & セキュリティ

### 3.1 暗号化標準

```
【転送中の暗号化】
プロトコル: TLS 1.3（強制）
暗号スイート: TLS_AES_256_GCM_SHA384
証明書: Let's Encrypt（自動更新）
HSTS: 有効（Strict-Transport-Security: max-age=31536000）

【保存中の暗号化】
DB暗号化: AES-256-GCM（Supabase標準）
ファイル暗号化: AES-256-XTS
キー管理: AWS KMS（暗号キーはAWS管理）

【パスワード保護】
ハッシュ関数: bcrypt + salt（14ラウンド）
レインボーテーブル対策: 自動salt生成
パスワード有効期限: 90日（変更強制）
```

### 3.2 アクセス制御（RLS: Row Level Security）

```
【Supabase RLS ポリシー】

生徒ユーザー:
  ├─ 自分のデータ: 読み取り・書き込み可
  ├─ 他の生徒のデータ: 読み取り不可
  ├─ クラス共通データ: 読み取り可
  └─ 講師の採点結果: 読み取り可

講師ユーザー:
  ├─ 自分のクラスのデータ: 全権限
  ├─ 他のクラスのデータ: 読み取り不可
  ├─ 全体ダッシュボード: 読み取り可
  └─ 管理者機能: 自分のクラスのみ

管理者:
  └─ すべてのデータ: 全権限
```

### 3.3 個人情報取扱方針

```
【取得する個人情報】
必須:
  - メールアドレス
  - パスワード（ハッシュ化後に保存）
  - 姓名
  - 学年

オプション:
  - 親の連絡先（保護者レポート送信用）
  - 生徒の学習データ（進捗追跡用）

【利用目的】
- ユーザー認証
- 学習進捗管理
- 成績レポート生成
- 親への報告書配信

【第三者提供】
なし（塾内でのみ利用）

【保持期間】
- 契約期間中: 保存
- 契約終了後:
  ・スコア・学習データ: 1年保持（バックアップ用）
  ・個人情報: 3ヶ月後に削除
  ・音声ファイル: 即削除

【GDPR対応】
- EUからのアクセス: 同意画面表示
- 削除要求: 30日以内に対応
- データ移行要求: XML/CSV形式で提供
```

### 3.4 定期監査

```
【セキュリティ監査】
频度: 年2回（上半期・下半期）
実施者: 外部セキュリティ企業（ISO 27001認定）
監査項目:
  ✓ ネットワークセキュリティ
```

## 🔟 FAQ（IT管理者向け）

### Q: ローカルネットワークのみで動作できますか？
```
A: いいえ。EigoMaster はクラウドサービスなため、
   インターネット接続が必須です。

   ただし、一度ダウンロードした音声ファイルはオフライン再生可能です。
```

### Q: 塾のデータはどこに保存されますか？
```
A: Supabase (AWS東京リージョン内) に保存されます。

   データセンター場所: 東京（ap-northeast-1）
   ジオレプリケーション: 大阪（同期）
   バックアップ: 別地域に保管

   日本国内で完全に保管されるため、
   個人情報保護法にも完全準拠しています。
```

### Q: 既存のLMS・教務システムと連携できますか？
```
A: 限定的に対応しています。

   対応済み:
   - GRADEUP (API連携)
   - Schoolmate (CSV import)

   その他のシステムについては、
   case-by-caseで対応を検討します。

   問い合わせ: partner@eigomaster.example.com
```

### Q: 万が一データが全て失われた場合の補償は？
```
A: 契約内容によりますが、一般的には以下の対応となります：

   1. 当社の過失: 回復可能なら復旧（通常1時間以内）
   2. ユーザーの操作誤り: 当社は復旧を試みますが保証なし
   3. 天災・テロ: 保証外

   詳細は保有する「サービス利用規約」第XX条をご参照ください。
```

### Q: 塾内でのデータ保管は可能ですか？
```
A: いいえ。EigoMaster はクラウドのみでの提供となります。

   理由:
   - セキュリティの維持
   - データの一元管理
   - バックアップの確実性
   - AI採点エンジンの稼働

   オンプレミス版の要望がある場合は、
   sales@eigomaster.example.com にご相談ください。
   （最小ユーザー数: 500人以上での検討対象）
```
