1. 実装内容概要
================================================================================

英検準1級向けのシャドーイング学習において、フレーズ単位（2-5語）での詳細な
フィードバック表示機能を実装しました。

【主な機能】
✅ フレーズごとのステータス表示（good/needsWork/major）
✅ カタカナ読み方の大きな表示
✅ 問題点の図解化（リンキング/リダクション/ストレス/イントネーション/発音）
✅ 改善のコツを「💡」ボックスで強調表示
✅ タップで詳細モーダル表示
✅ 段階的な学習フロー（フレーズ → 詳細情報）

3. UI/UX デザイン
================================================================================

【ステータス別色分け】
✅ good:       緑 (#52C41A)
⚠️ needsWork:  黄 (#FAAD14)
❌ major:      赤 (#F5222D)

【Issue Type アイコン】
🔗 linking      - 単語間の連結
⬇️ reduction    - 音の削減（リダクション）
💪 stress      - アクセント・強弱
📈 intonation  - イントネーション
🗣️ pronunciation - 個々の音の発音

【画面遷移フロー】
ShadowingResultScreen
  └→ 「📝 詳細な分析を見る」
     └→ DetailedShadowingFeedback
        ├→ 「🎯 フレーズ単位の分析」セクション
        │  └→ フレーズカード（複数）
        │     └→ タップ
        │        └→ PhraseDetailModal（全詳細表示）
        │
        └→ 「📝 各単語の評価」セクション（wordFeedbacks 時）

4. データフロー
================================================================================

Claude API
  ↓
scoreShaddowingRecording() → phraseFeedbacks 生成
  ↓
ShadowingScreen addRecord()
  ↓
Zustand useShadowingStore (phraseFeedbacks 保存)
  ↓
Supabase sync (JSON stringify)
  ↓
ShadowingResultScreen
  ↓
DetailedShadowingFeedback
  ↓
PhraseDetailModal (フレーズ詳細)

7. 重要な実装ポイント
================================================================================

【TypeScript 型安全性】
- PhraseFeedback インターフェース定義済み
- WordFeedback インターフェース定義済み
- ShadowingRecord に両方を含める
- JSON 操作時は try-catch でエラーハンドリング

【色分けロジック】
- getPhraseStatusColor(): ボーダーカラー
- getPhraseStatusBgColor(): 背景カラー
- ステータスごとに rgba(r, g, b, 0.1) で統一感

【モーダル表示】
- PhraseDetailModal: フルスクリーン表示
- SafeAreaView でセーフエリア対応
- scrollView でスクロール対応

【ストア同期】
- phraseFeedbacks/wordFeedbacks は JSON 保存
- parse 時に type guard で型安全性確保
- AsyncStorage キャッシュ対応
